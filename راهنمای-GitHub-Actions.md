# ساخت خودکار APK با GitHub Actions (بدون نصب هیچ‌چیزی روی کامپیوتر خودتان)

این پروژه یک فایل آماده دارد: `.github/workflows/build-apk.yml`
با هر بار push شدن این پروژه به گیت‌هاب، خودِ سرورهای گیت‌هاب (رایگان) این کارها را
انجام می‌دهند: نصب Node و Java، اضافه‌کردن پلتفرم اندروید با Capacitor، و ساخت APK —
و در پایان یک فایل APK قابل‌دانلود تحویل می‌دهند.

## مراحل (حدود ۵ دقیقه)

### ۱. یک ریپازیتوری تازه در GitHub بسازید
به github.com بروید → **New repository** → یک اسم بدهید (مثلاً `kurdish-conjugator`) →
**Create repository** (نیازی نیست چیزی تیک بزنید، خالی بسازید).

### ۲. محتوای این پوشه را در همان ریپو push کنید
داخل همین پوشه (`kurdish-app`) در ترمینال:
```bash
git init
git add .
git commit -m "اولین نسخه"
git branch -M main
git remote add origin https://github.com/USERNAME/kurdish-conjugator.git
git push -u origin main
```
به‌جای `USERNAME` نام کاربری گیت‌هاب خودتان را بگذارید.

### ۳. اجرای خودکار را تماشا کنید
در صفحه‌ی ریپو در گیت‌هاب، برو به تب **Actions** — می‌بینید که workflow با نام
«Build APK» شروع به اجرا کرده (حدود ۲ تا ۴ دقیقه طول می‌کشد).

### ۴. دانلود APK
وقتی اجرا با ✅ سبز تمام شد، روی همان اجرا کلیک کنید؛ پایین صفحه بخش
**Artifacts** هست — فایلی با نام `kurdish-conjugator-debug-apk` را دانلود کنید.
داخلش `app-debug.apk` است — همین را روی گوشی اندروید خود منتقل و نصب کنید
(باید «نصب از منابع ناشناس» را در گوشی فعال کنید).

## اگر بعداً بخواهید اپ را عوض کنید
فقط کافی‌ست `www/index.html` را ویرایش کنید، دوباره commit/push کنید — workflow
خودش دوباره اجرا می‌شود و APK جدید می‌سازد. نیازی به اجرای دستی هیچ دستوری نیست.

## نکته
این نسخه‌ی APK یک نسخه‌ی «debug» است — برای نصب شخصی و تست کاملاً کافی است.
اگر بعداً خواستید در Google Play منتشرش کنید، باید نسخه‌ی امضاشده (release/signed)
بسازید که نیاز به یک کلید امضا دارد؛ آن مرحله را هروقت خواستید می‌توانم برایتان
به همین workflow اضافه کنم.
